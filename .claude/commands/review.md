# /review — 代码审查

审查代码质量，发现问题并提供改进建议。

## 使用方式

```
/review                      # 审查当前 git diff
/review page_objects/        # 审查指定目录
/review test_search.py       # 审查指定文件
```

## 审查维度

- 代码风格：命名、格式、一致性
- 代码质量：重复、复杂度、可读性
- 安全检查：注入、泄露、权限
- 潜在问题：边界条件、异常处理

## 输出格式

```markdown
## 代码审查报告

### 🔴 严重问题
- [文件:行号] 问题描述

### 🟡 警告
- [文件:行号] 问题描述

### 🔵 建议
- [文件:行号] 改进建议
```
