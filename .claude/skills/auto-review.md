# Auto Review Skill

## 触发条件
当代码被修改、新增或删除时自动触发。

## 功能
自动审查代码质量。

## 执行流程
1. 检测代码变更
2. 分析代码质量
3. 检查是否符合规范
4. 提供改进建议

## 审查维度
1. **代码风格** — 命名规范、格式规范
2. **代码质量** — 函数长度、职责单一
3. **安全性** — 硬编码凭证、敏感信息
4. **文档完整性** — docstring、注释

## 输出
审查结果应包含：
- 严重问题（必须修复）
- 警告信息（建议修复）
- 改进建议（可选优化）

## 自动触发场景
- 文件保存后
- Git commit 前
- 代码重构后
