# Dependency Check Skill

## 触发条件
当 requirements.txt 被修改时自动触发。

## 功能
检查依赖安全性。

## 执行流程
1. 读取 requirements.txt
2. 检查已知漏洞
3. 检查未使用的依赖
4. 提供安全建议

## 检查内容
1. **安全漏洞** — 检查是否有已知的安全漏洞
2. **版本过旧** — 检查是否有更新的版本
3. **未使用依赖** — 检查是否有未使用的包
4. **依赖冲突** — 检查是否有版本冲突

## 输出
检查结果应包含：
- 安全漏洞列表
- 版本更新建议
- 未使用依赖列表
- 依赖冲突列表

## 工具
- pip-audit — 检查安全漏洞
- pip-autoremove — 检查未使用依赖
